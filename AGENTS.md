# AGENTS.md - OaboutAI Hugo Knowledge Archive

Authoritative operating manual for AI agents working in this repository.

## 1. Operating Model

This repository is a composable Hugo monorepo:
- `core/`: reusable framework (layouts, assets, scripts, theme)
- `apps/<app-id>/`: app-level content and configuration
- Default app: `oaboutai`

Non-negotiable principles:
1. Every entry must be source-traceable.
2. Metadata is a hard contract.
3. Topic taxonomy is controlled (max 10).
4. Keyword taxonomy is controlled.
5. English canonical entry is required for each slug.
6. OpenClaw ingestion requires paired `en` + `zh-tw` in the same run.

## 2. Canonical Paths

Inside `apps/<app-id>/`:
- `content/en/items/<slug>/index.md`
- `content/zh-tw/items/<slug>/index.md`
- `content/<lang>/topics/<topic>.md` (generated by `scripts/sync_topics.py`)
- `content/<lang>/archive/monthly.md`
- `data/topics.json`
- `data/keywords.json`
- `data/keyword_proposals.jsonl`
- `app.toml`

Attachment rules:
- Store attachment files in the same bundle as `index.md`.
- Front matter `attachments` uses relative filenames only.
- Copyright-safe mode: keep risky originals external and link via `archived_url`.

## 3. Required Metadata

Every item front matter must include:
- `title`
- `source_url`
- `source_type` (`webpage|pdf|youtube|other`)
- `types` (single-item array equal to `source_type`)
- `source_date` (`YYYY-MM-DD`)
- `submission_date` (`YYYY-MM-DD`)
- `executive_summary`
- `detailed_notes`
- `keywords` (array)
- `primary_topic` (single topic id)
- `topics` (array of secondary topic ids; exclude `primary_topic`)
- `language` (`en|zh-tw`)

Optional:
- `attachments`, `authors`, `publisher`, `archived_url`, `duration`

## 4. Taxonomy Governance

Topics:
- Source of truth: `apps/<app-id>/data/topics.json`
- Max 10 top-level topics

Keywords:
- Source of truth: `apps/<app-id>/data/keywords.json`
- If no exact keyword exists:
  1. map to closest existing id
  2. append proposal line to `apps/<app-id>/data/keyword_proposals.jsonl`
- Never invent keyword IDs in entries.

## 5. Slug and Date Rules

Slug format:
- `YYYYMMDD-short-kebab-title`

Date semantics:
- `source_date`: source published/captured date
- `submission_date`: archive ingest date
- both must be ISO `YYYY-MM-DD`

## 6. OpenClaw Workflow (Authoritative)

References:
- `docs/openclaw_ingestion_workflow.md`
- `docs/openclaw_system_prompt.md`
- `scripts/ingest_item.py`

Procedure:
1. Prepare draft:
   - `python scripts/ingest_item.py prepare --source-input "<url-or-local-path>" --source-date "YYYY-MM-DD" --output /tmp/oaboutai_draft.json`
2. Fill bilingual fields and taxonomy IDs in draft JSON.
3. Dry run:
   - `python scripts/ingest_item.py ingest --spec-file /tmp/oaboutai_draft.json --dry-run`
4. Write + checks:
   - `python scripts/ingest_item.py ingest --spec-file /tmp/oaboutai_draft.json --run-checks`
5. Build guard (mirrors CI):
   - `python scripts/compose_site.py --app-id "${APP_ID:-oaboutai}" --output /tmp/oaboutai-site --clean`
   - `cd /tmp/oaboutai-site`
   - `python scripts/sync_topics.py`
   - `rm -f data/keyword_proposals.jsonl`
   - `npx --yes hugo-bin --gc --minify`
6. Optional direct push:
   - `python scripts/ingest_item.py ingest --spec-file /tmp/oaboutai_draft.json --run-checks --git-push`

Expected outputs:
- `apps/<app-id>/content/en/items/<slug>/index.md`
- `apps/<app-id>/content/zh-tw/items/<slug>/index.md`
- attachments in EN bundle
- optional proposals in app proposal queue

## 7. Maintenance

Add topic:
1. Edit `apps/<app-id>/data/topics.json`
2. Keep total topics <= 10
3. Run `python scripts/sync_topics.py`

Add keyword:
1. Edit `apps/<app-id>/data/keywords.json`
2. Backfill existing entries if needed
3. Clean processed lines in `apps/<app-id>/data/keyword_proposals.jsonl`

## 8. Quality Gate

CI workflow `.github/workflows/docs-site-ci.yml` must pass:
1. compose + topic sync
2. metadata checks
3. Hugo build

Do not bypass CI for content updates.

## 9. Anti-Patterns

- Missing required front matter
- Unknown topic/keyword IDs in entries
- Attachments outside bundle
- zh-TW-only entries without EN canonical counterpart
